Regular expression (regex, regexp)

formal language used in computer programs that work with text to find and manipulate substrings in the text

Usually such patterns are used by string-searching algorithms for "find" or "find and replace" operations on strings, or for input validation.

In JavaScript, regular expressions are also objects

These patterns are used by methods of RegExp and String objects.

How to create:

  • using a regular expression literal:

    var re = /ab+c/;

  • calling the constructor function of the RegExp object:

    var re = new RegExp("ab+c");

How to use?

Simple pattern

simple patterns are constructed of characters for which you want to find a direct match.

Example

  • Text: Sea, sea, ocean
  • RegEx: sea
  • Result: sea

Metacharcacters

  • Used for expand functionality of simple patterns

    • . (dot) - search for any character

    Example

    • Text: Anya Asya Olya Alya Valya
    • RegEx: A.ya
    • Result: Anya Asya Alya

    The dot is a very convenient symbol, but at the same time a very dangerous symbol - if you use it, be sure to test the resulting regular expression

    Example

    • Text: А7ya А&ya Olya А ya Valya A.ya
    • RegEx: A.ya
    • Result: А7ya А&ya А ya A.ya

    A dot will also find a dot! But if we want exactly a dot, we need to shield it, i.e. add a backslash before it:

    Example

    • RegEx: .txt
    • Result: file.txt log.txt one_txt.jpg

    Example

    • RegEx: \.txt
    • Result: file.txt log.txt

    [] - range of values allowed

    Example

    • Text: Anna Alla Arkan A74alrt
    • RegEx: A[nn][ll]a
    • Result: Anna Alla
    • [нл] — only russian «н» и «л»
    • [а-я] — lowercase russian letters (except "ё)
    • [А-Я] — uppercase russian letters
    • [А-Яа-яЁё] — any russian letters
    • [a-z] — latin alphabet
    • [a-zA-Z] — any latin letters
    • [0-9] — any number
    • [a-gx-z] — range from a to g and from x to z

    ^ - exception

    used within the range of values

    • [^0-9] — any letters except numbers
    • [^ёЁ] — any letters except russian "ё"
    • [^a-c8] — any letters except "a", "b", "c" and 8

    Example

    • Input: file.txt file.txt file_1.txt 1.txt
    • RegEx: [^0-9]\.txt
    • Result: file.txt file.txt

    Example

    • Input: fruits[0] fruits0
    • RegEx: fruits\[0\]
    • Result: fruits[0]

    Example

    • Input: arr
    • RegEx: fruits\[[0-9]\]
    • Result: fruits[0] = “orange”; fruits[1] = “apple”; fruits[2] = “lemon”;

    | - list of options

    like a range of values, but are better suited to list the words

    Example

    • Input: Olya Olga Cat Kitty
    • RegEx: Olya|Cat
    • Result: Olya Cat

    Example

    • RegEx: A(nn/ll)a
    • Result: Anna Alla

    Metacharacters

    can be used to replace the range of values

    Symbol Equivalent Description
    \d [0-9] Number
    \D [^0-9] Non-number
    \s [ \f\n\r\t\v] Space-symbol
    \S [^ \f\n\r\t\v] Non-space-symbol
    \w [[:word:]] Letter, number or _
    \W [^[:word:]] Any except letter, number or _

    Quantifires

    specify the number of times the text is repeated

    Quantifire Repeat times
    ? 0-1
    * 0 and more
    + 1 and more

    Example

    • RegEx: .*\d\d\.\d\d\.\d\d\d\d.*
    • Result: 01.01.2000

    Advanced quantifires:

    Quantifire Repeat times
    {n} exactly n
    {m,n} from m to n inclusive
    {m,} At least m
    {,n} Not more than n

    Example

    • RegEx: .*\d{2}\.\d{2}.\d{4}*
    • Result: 01.01.2000

    Flags

    - allow for functionality like global searching and case-insensitive searching. These flags can be used separately or together in any order, and are included as part of the regular expression.
